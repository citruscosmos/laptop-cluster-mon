## Architecture

```
[Proxmox Node 1]───node_exporter:9100────┐
[Proxmox Node 2]───node_exporter:9100────┤
[Proxmox Node 3]───node_exporter:9100────┤
                                          ├──Prometheus (host net)──Grafana
[Monitoring VM]                          │
  ┌──────────────────────────────────┐   │
  │ Docker Compose                   │   │
  │  - prometheus (network_mode:host)│◄──┘
  │  - grafana (bridge, :3000)       │
  │  - pve-exporter (bridge, :9221)  │────Proxmox API (:8006)
  └──────────────────────────────────┘
```

**Networking:**
- Prometheus uses `network_mode: host` to reach Node Exporter on Proxmox hosts (:9100) and pve-exporter via localhost:9221
- Grafana on bridge network, port 3000 published
- pve-exporter on bridge network, port 9221 published (reachable from Prometheus at localhost:9222)

## Implementation Steps

### Step 1: Template files
Create static template files with `__PLACEHOLDER__` markers for envsubst substitution:
- `templates/prometheus.yml.tmpl` — 3 scrape jobs (pve_cluster, pve_nodes, node_exporter), configurable scrape_interval via `__SCRAPE_INTERVAL__`, node IPs via `__NODE_IPS_YAML__`
- `templates/pve.yml.tmpl` — Proxmox API credentials via `__PVE_API_TOKEN__`
- `templates/docker-compose.yml.tmpl` — 3 services, Prometheus with `network_mode: host`, restart: unless-stopped on all
- `templates/datasource.yml.tmpl` — Grafana Prometheus datasource with explicit UID `__DATASOURCE_UID__`

### Step 2: setup.sh
Flow:
```
1. root check + OS check (Debian/Ubuntu)
2. Pre-flight checks:
   - jq installed (apt install if missing)
   - docker compose available (standalone or plugin)
   - curl can reach PVE_NODE_IP:8006
   - API token authenticates (curl /api2/json/access/permissions → 200)
3. Docker install (skip if exists, add user to docker group)
4. Env var validation (PVE_NODE_IP, PVE_API_TOKEN required)
5. Cluster auto-discovery:
   - curl /api2/json/cluster/status
   - Parse JSON with jq: .data[].ip
   - Validate: at least 1 IP returned (warn if only 1, this is the known node)
   - Fallback: if API fails, use PVE_NODE_IP alone with warning
6. Generate configs from templates using envsubst:
   - Substitute __NODE_IPS_YAML__ with YAML-formatted IP list
   - Substitute __PVE_API_TOKEN__, __SCRAPE_INTERVAL__, __DATASOURCE_UID__
7. Copy Grafana dashboard JSONs to provisioning/dashboards/
8. docker compose up -d
9. Post-setup validation:
   - Wait 5s for pve-exporter to start
   - curl localhost:9221/pve?target=<node-ip>&module=default | grep 'pve_version_info'
   - Verify at least one pve_* metric has non-zero value
10. Completion message with URLs, Node Exporter install instructions, disk usage estimate
```

## Key Decisions from Review

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | Prometheus uses host networking | Required to reach Node Exporter on Proxmox hosts; also simplifies pve-exporter access via localhost |
| D2 | envsubst for template substitution | Avoids sed corruption of API tokens containing / or & |
| D3 | restart: unless-stopped on all services | Stack survives VM reboots |
| D4 | Pre-downloaded dashboard JSONs | Reproducible, no network dependency at setup time |
| D5 | Container image tags pinned | Prevents silent breakage when latest tag changes metric names |
| D6 | Pre-flight checks before config generation | Fail fast with clear error messages instead of generating broken configs |
| D7 | Post-setup metrics validation | Verify pve-exporter returns actual Proxmox metrics, not just HTTP 200 |
| D8 | VM sizing: 4GB RAM, 2 vCPU, 20GB disk | Prometheus 1-2GB for 3 nodes at 15s scrape, headroom for 15-day retention |
| D9 | Configurable scrape interval (env var, default 15s) | Users with many VMs can reduce Prometheus load |
| D10 | Grafana datasource UID explicitly set | Ensures dashboard JSONs reference the correct datasource |
| D11 | Grafana admin password configurable via env var | Security — default admin:admin but can be overridden |

## NOT in scope

| Item | Rationale |
|------|-----------|
| Node Exporter installation on Proxmox hosts | Manual install; script prints curl + systemd instructions |
| Monitoring VM provisioning | User creates VM manually or via Proxmox API — out of setup script scope |
| Alerting (Alertmanager, webhooks) | Out of MVP scope; can be added later via docker-compose override |
| Custom cluster map dashboard (Approach B) | Deferred; standard dashboards ship first, custom dashboard as follow-up |
| Periodic IP rediscovery | Proxmox nodes expected to have static IPs in homelab; document as known limitation |
| Disk space guard (auto-stop Prometheus at 95%) | Acknowledged as future improvement; 15d retention + 20GB disk provides safe margin for 3-node cluster |
| Rollback on script failure | Acknowledged as known limitation; user re-runs script after fixing the issue |
| HTTPS/TLS for Grafana and Prometheus | Homelab — plain HTTP is acceptable |

## Failure Modes

| # | Failure | Test? | Error handling? | User sees? |
|---|---------|-------|-----------------|------------|
| 1 | API token invalid → pve-exporter returns error metrics | Post-setup validation catches this | Yes — validation curl fails with clear message | Clear error: "pve-exporter returned no Proxmox metrics — check API token" |
| 2 | Prometheus can't reach pve-exporter (port conflict) | docker compose ps check | docker compose ps output shown | Service status shown |
| 3 | Cluster not quorate → auto-discovery returns empty | jq parse validation | Fallback to single known node with warning | Warning: "Only 1 node detected — using provided IP. Verify cluster quorum." |
| 4 | jq not installed → JSON parsing fails | Pre-flight check | Script installs jq via apt | User sees "Installing jq..." |
| 5 | Token contains special chars → mangled by sed | N/A (envsubst used) | envsubst handles all chars safely | Clean config generated |
| 6 | Grafana datasource name mismatch → "Datasource not found" | N/A (UID pinned) | Explicit UID in provisioning | Dashboards load with data |
| 7 | Prometheus disk fills up → silent failure | Not in MVP | Not in MVP | **Critical gap** — Prometheus stops ingesting, Grafana goes blank. Mitigation: 20GB disk + 15d retention is safe for 3-node cluster. Future: add disk monitor. |
